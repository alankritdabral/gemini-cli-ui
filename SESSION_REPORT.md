## 🛠️ Bugs & Blockers Encountered

### 1. Browser Agent "X Server" Dependency
*   **Issue**: The `@browser_agent` (using Playwright/Puppeteer) defaults to a "headful" mode that requires an X11 display server. In this terminal environment, this causes a fatal crash: `Missing X server to start the headful browser`.
*   **Attempted Fixes**:
    *   Set `headless: true` in `settings.json` (Ignored by the agent in this version).
    *   Created system-level symlinks/wrappers for `google-chrome` and `chrome` pointing to `chromium`.
    *   Surgically patched the CLI source code in `node_modules` and `npx` cache to force `--headless` flags and disable X server checks.
*   **Result**: **Partial Failure**. Even with code patches, the agent architecture in this specific CLI version seems to have hardcoded display requirements that are difficult to bypass without `xvfb`.

### 2. Browser Executable Path
*   **Issue**: The CLI looks for Chrome at `/opt/google/chrome/chrome`, but the system only has Chromium installed via Snap at `/snap/bin/chromium`.
*   **Fix**: Created a wrapper script at `~/.local/bin/google-chrome` that redirects calls to the Snap Chromium with the `--headless` flag injected.

### 3. UI Event Bubbling
*   **Issue**: Clicking "History" inside the 3-dot menu triggered a global window listener that immediately closed the menu before the dropdown could show.
*   **Fix**: Added `event.stopPropagation()` to the `moreHistoryButton` listener in `webview/main.js`.

## 💡 Efficient Workarounds (The "Golden Path")

### 🌐 Web Data Retrieval
*   **Don't use**: `@browser_agent` for simple data scraping in this environment (due to the X server issue).
*   **Use instead**: `web_fetch`. It is faster, bypasses the display server requirement entirely, and successfully retrieves data from authenticated sites like LinkedIn when they have public-facing job boards.

### 🤖 Agent Development
*   **Location**: Always develop your agents in `.gemini/agents/` within the project.
*   **Mechanism**: The new `syncExtensionAgents` function in `src/extension.ts` handles the heavy lifting of moving them to the global scope.
